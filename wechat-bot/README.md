# 📱 个人微信机器人使用指南

> ⚠️ **风险提示**：个人微信机器人存在封号风险，建议用小号测试！

---

## 🚀 快速启动

### 1. 一键配置
```bash
cd /home/user/.openclaw/workspace
./setup_wechat_bot.sh
```

### 2. 编辑配置
```bash
# 修改配置文件，添加要监控的群聊
nano wechat-bot/config.py
```

配置示例：
```python
# 要监控的群聊名称（部分匹配）
MONITOR_GROUPS = [
    "工作群",
    "产品技术群", 
    "家庭群",
]

# 触发关键词
TRIGGER_KEYWORDS = [
    "总结",
    "summary",
    "汇总",
    "@助手",
]

# 自动日报时间（晚上8点）
AUTO_REPORT_HOUR = 20
```

### 3. 启动机器人
```bash
cd wechat-bot
python3 wechat_summarizer.py
```

### 4. 扫码登录
- 终端会显示二维码
- 用微信小号扫码登录
- 登录成功后开始监控

---

## 📱 使用方法

### 方式1：关键词触发
在监控的群聊中发送：
```
总结
summary  
汇总
@助手
```

机器人会自动回复群聊总结。

### 方式2：自动日报
每天晚上8点（可配置），自动生成并发送日报。

### 方式3：私聊查询
私聊机器人发送：
```
总结 [群名]
例如：总结 工作群
```

## ⚠️ 重要注意事项

### 封号风险
- **概率**：中等（约20-30%使用者在1个月内被封）
- **原因**：微信检测异常登录、频繁消息
- **建议**：
  - ✅ 使用小号
  - ✅ 控制发送频率
  - ✅ 不要24小时不间断运行
  - ❌ 不要用主号
  - ❌ 不要频繁@所有人

### 稳定性问题
- 网页版微信可能随时掉线
- 需要定期重新扫码登录
- 复杂消息（语音、视频）可能无法处理

### 安全建议
1. 使用独立小号，不绑定重要信息
2. 不要在群聊中透露机器人身份
3. 定期检查登录状态
4. 发现异常立即停用

## 🐛 常见问题

### Q1: 扫码后显示"登录失败"
**A**: 
- 微信网页版可能被限制
- 尝试用企业微信或个人微信小号
- 检查网络连接

### Q2: 机器人不回复消息
**A**:
- 检查是否在监控列表中
- 检查触发词是否正确
- 查看终端是否有错误日志

### Q3: 如何退出登录？
**A**:
- 按 `Ctrl+C` 停止程序
- 删除 `itchat.pkl` 文件
- 重新运行即可重新扫码

### Q4: 可以同时监控多个群吗？
**A**: 可以，在 `MONITOR_GROUPS` 中添加多个群名。
